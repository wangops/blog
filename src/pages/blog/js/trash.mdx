#### Javascript 垃圾回收机制
###### &nbsp;
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;JavaScript的垃圾回收机制是一种自动内存管理机制，旨在帮助开发人员管理内存并避免内存泄漏。
###### &nbsp;
#### 工作原理
###### &nbsp;
####  1 标记清除（Mark-and-Sweep）：
###### &nbsp;

&nbsp;&nbsp;&nbsp;&nbsp;  标记阶段：从根对象（通常是全局对象或者由代码中定义的对象）开始遍历所有引用的对象，并标记它们为“活动”（即被引用）。
###### &nbsp;
&nbsp;&nbsp;&nbsp;&nbsp;  清除阶段：遍历整个堆（所有内存空间），清除那些未被标记为“活动”的对象，这些对象被视为“垃圾”，可以被安全地回收。
###### &nbsp;

####  2 引用计数：
###### &nbsp;
&nbsp;&nbsp;&nbsp;&nbsp; 在过去，某些浏览器和JavaScript引擎使用引用计数来跟踪对象的引用数。当对象的引用数为零时，对象被视为不再需要，可以被垃圾回收。
###### &nbsp;
&nbsp;&nbsp;&nbsp;&nbsp; 但引用计数方法难以处理循环引用的情况（例如对象A引用对象B，对象B也引用对象A），因此在现代浏览器和引擎中已不再主流。
